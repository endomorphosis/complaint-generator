Intake Questions

Capture the facts once, then reuse them everywhere else in the workflow. The optional court-caption field helps the complaint open with a more realistic court header when you know the right district.

What to do here Focus on who was involved, what happened, when it happened, and what harm followed. If the exact date or wording is uncertain, save the best current version and improve it later.

Start with the story the complainant can tell clearly today

Use plain language. If dates or exact wording are uncertain, capture the best current version and come back with corroboration in the Evidence stage.

Ask for sequence What happened first, what happened next, and how soon after the protected activity did the harm occur?
Name the actors Capture who made the decision, who knew about the complaint, and who communicated the adverse action.
Leave room for uncertainty Approximate dates and incomplete recall are acceptable at intake if they are marked for later corroboration. 
Caption preview will appear here as the court and party fields are filled in.
Nothing is filed from this page automatically. Saving here only updates the shared complaint workspace.

Evidence Intake

Collect both testimony and documents, and attach each item to a claim element so support gaps are obvious.

What to do here Add the records or witness details most likely to strengthen the missing parts of the complaint. Every saved item should answer the question: what does this help prove?

What counts as helpful support

Prefer concrete sources: emails, messages, witness accounts, meeting notes, timeline records, HR filings, performance reviews, and anything that helps anchor protected activity, employer knowledge, causation, or damages.

Support-gap workbench Use this area to decide what kind of proof would move the complaint forward most. If the case already has a timeline but weak corroboration, add documents or witness support before drafting.

Draft and Edit

Generate the complaint from the workspace state, then make final edits without leaving the same flow.

What to do here Generate a first draft once the timeline, protected activity, adverse action, and harm feel stable. Then refine clarity, tone, and requested relief before treating it as final.

Draft when the case theory is coherent

A usable draft needs a stable plaintiff identity, an opposing party, protected activity, adverse action, timing, and harm. If one of those pieces is weak, use Review and Evidence first.

Filing editor checklist Before treating the draft as usable, make sure it names the parties clearly, ties the adverse action to the protected activity, describes the harm, and asks for relief that matches the record.

CLI and MCP Tools

The website is using the JavaScript MCP SDK below. The same service is also available through a Python CLI and a lightweight stdio MCP tool server.

Python package imports

from complaint_generator import ComplaintWorkspaceService, create_review_surface_app, export_complaint_docx
from complaint_generator.mcp import handle_jsonrpc_message

service = ComplaintWorkspaceService()
session = service.get_session("did:key:example")
docx_artifact = export_complaint_docx("did:key:example", service=service)

Python CLI

complaint-workspace session --user-id did:key:example
complaint-generator-workspace session --user-id did:key:example
complaint-workspace answer --user-id did:key:example --question-id protected_activity --answer-text "Reported discrimination to HR"
complaint-generator-workspace generate --user-id did:key:example --requested-relief "Back pay|Injunctive relief"
complaint-workspace export-markdown --user-id did:key:example
complaint-workspace export-docx --user-id did:key:example
complaint-workspace export-pdf --user-id did:key:example
complaint-workspace browser-audit artifacts/ui-audit/browser-audit

MCP stdio server

complaint-generator-mcp
complaint-mcp-server
tool: complaint.export_complaint_docx
tool: complaint.run_browser_audit

JavaScript MCP SDK

const client = new window.ComplaintMcpSdk.ComplaintMcpClient();
const bootstrap = await client.bootstrapWorkspace();
await client.callTool('complaint.review_case', { user_id: bootstrap.did });
await client.exportComplaintMarkdown(bootstrap.did);
await client.exportComplaintDocx(bootstrap.did);
await client.exportComplaintPdf(bootstrap.did);
await client.runBrowserAudit({ screenshot_dir: 'artifacts/ui-audit/browser-audit' });
